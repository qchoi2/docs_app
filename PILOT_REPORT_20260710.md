## 체크리스트 판정

### 1. 미분류가 많은가? — 통과
미분류 2/271건 (0.7%). 둘 다 "Vienna - 투자계약_*" — type_rules에 투자계약 패턴이
없어서다. manual_overrides 예시로 처리 가능:
`"**/*투자계약*": { ctype: 투자계약 }` 또는 type_rules에 규칙 추가(소유자 결정).

### 2. 국문/영문/국영문 분류가 맞는가? — 버그 발견·수정 후 통과
점검 결과 영문 분류 43건 **전부** 한글비율 88~99%의 국문 계약서였다.
원인: lang 규칙이 경로+본문 전체를 매칭해, 국문 계약서의 언어조항
("국문과 영문...", "English")에 걸림. **lang 분류를 경로 신호만 쓰도록 수정**하고
(ctype과 동일 원칙, 커밋 ad2ed4e) DB 재분류 실행.
수정 후: 국문 263 / 영문 2(실측 한글비율 0.00 — 진짜 영문) / 미상 6(스캔 PDF).
국영문 0건은 이 샘플에 병기 문서가 없어서로 보임 — 전체 코퍼스에서 재확인 필요.

### 3. 드래프트 제외가 정상인가? — 통과
is_draft 분포: draft 170 / 체결본 79 / 판별불가 22.
"손해배상" 262건 → `--exclude-drafts` 후 92건. 남은 결과에 draft 0건,
판별불가 21건 포함(스펙대로 포함+표기). 드래프트가 170건(63%)인 것은
markup/초안 파일명이 많은 이 코퍼스 특성과 일치.

### 4. 중복 제거가 과하게 되지는 않았는가? — 통과 (과소 쪽 한계 있음)

### 6. 2글자 검색어 warning이 적절히 나오는가? — 통과

`short_term_fallback:<term>` warning 정상 기록. 조용한 0건 없음.

### 7. 검색 속도가 감당 가능한가? — 통과 (조건부)
로컬 디스크 기준 확장 검색 0.4~1.4초/질의(FTS+동의어 7~11변이),
2글자 LIKE 폴백 0.3~0.4초, 복합(타입+언어+AND 2키워드) 0.5초.
사양이 더 좋은 실제 PC 로컬 NVMe에서는 이보다 빠를 것. 단, cs_index를
네트워크 드라이브에 두면 수 배 느려짐 — 반드시 로컬 디스크(README 경고).
